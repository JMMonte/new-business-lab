Design the validation experiments for the idea in `businesses/$1/` using the
`validator` agent (follow `${CLAUDE_PLUGIN_ROOT}/frameworks/06-test.md`).

Aim the first experiment at the dominant assumption (load-bearing from Think, top
risk from Audit, most sensitive from Simulate). Order experiments by information
value / cost. Write each as a falsifiable hypothesis with **pre-committed**
success and kill thresholds, plus cost and time. Climb the validation ladder
(Problem → Demand → Solution → Willingness-to-pay → Channel → Retention) only as
far as needed.

Write `06-experiment-plan.md` with the prioritized list, Experiment 1 in
run-ready detail, a decision tree, and total budget/timeline to a build-or-kill
decision. Reject vanity metrics — measure behavior with a cost attached.

Finish with the first experiment to run this week, its cost, and the result that
would kill the idea.
